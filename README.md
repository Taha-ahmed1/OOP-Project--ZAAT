# CAR MARKETPLACE SYSTEM (OOP PROJECT)

## Project Title
Car Marketplace System (ZAAT Luxury Autos)

## Project Description:
This project is a simple car marketplace system where users can act as Admin, Buyer, or Seller.
The system allows adding car listings, approving listings, buying cars, saving favorites, and managing marketplace data using OOP concepts like inheritance, polymorphism, abstraction, and operator overloading.

## Use Cases:
1. Admin can approve or remove car listings
2. Admin can view all available listings
3. Seller can add new car listings
4. Seller can update or delete listings
5. Buyer can view available cars
6. Buyer can save favorite cars
7. Buyer can purchase a car
8. System saves and loads listings from file (CSV)

## How to Run

After compiling, choose:
1) Admin
2) Buyer
3) Seller

Then you can select options for the selected user for specific actions.

## How to Use System

* Admin approves listings before they appear to buyers
* Seller adds new cars to marketplace
* Buyer can browse and purchase cars
* Favorites can be saved for quick access
* Listings are shown only if approved and not sold


## Assumptions / Limitations

* Maximum 20 listings allowed
* Maximum 5 favorites per buyer
* No database used (only file handling with CSV)
* No login system implemented (simple role selection only)
* Memory is not fully optimized (basic student-level implementation)
